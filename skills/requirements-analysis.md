# Skill: Requirements Analysis

## Objetivo

Transformar descrições em requisitos estruturados.

## Saída esperada

* Requisitos funcionais
* Regras de negócio
* Entidades
* Casos de uso

## Regras

* Não inventar requisitos
* Manter consistência com o domínio
* Usar linguagem clara

## Exemplo

Input: "Sistema de alunos"

Output:

* Criar aluno
* Editar aluno
* Listar alunos
