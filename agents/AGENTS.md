# AGENTS.md

## Project Context

Web system for managing students, classes, and evaluations.

Required stack:

* Frontend: React + TypeScript
* Backend: Node.js + TypeScript
* Tests: Cucumber (Gherkin)
* Persistence: JSON

## Important Rules

* Code must be simple, modular, and readable
* Always separate concerns (controller, service, repository)
* Never mix business rules with the presentation layer
* Strong typing always (avoid `any`)

## Development Flow

1. Create or update acceptance tests (Gherkin)
2. Implement backend
3. Implement frontend
4. Validate scenarios

## Available Skills

* `requirements-analysis` — extract and structure requirements from documents
* `backend-ddd-typescript` — scaffold backend modules with controller/service/repository
* `react-frontend-crud` — scaffold React CRUD pages with TypeScript
* `cucumber-acceptance` — write Gherkin feature files and step definitions
* `json-persistence` — implement file-based JSON repositories
* `email-digest` — implement daily email consolidation logic

## Conventions

* Use camelCase
* Write all code in English
* Avoid duplication
* Prefer pure functions
